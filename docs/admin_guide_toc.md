### 📌 Chapter 1. 설치 및 초기 구성

#### [Slide 03] 변환 엔진 설치 및 경로 설정
*   **슬라이드 제목**: 1.1. 설치 패키지 압축 해제 및 경로 구성
*   **필수 텍스트**:
    *   배포된 Zip 압축 파일을 풀어 반드시 `C:/IPLMS` 루트 디렉토리에 설치합니다.
    *   **권장 폴더 구조**:
        *   `C:/IPLMS/bin` : 실행 파일 및 실행 배치 파일 (`IPLMSDocsDaemon.cmd`) 위치
        *   `C:/IPLMS/properties/config.properties` : 통합 엔진 설정 파일
        *   `C:/IPLMS/lib` : 종속성 자바 라이브러리 폴더
        *   `C:/IPLMS/91_input` : 변환 대상 원본 파일 적재 폴더 (예시)
        *   `C:/IPLMS/92_output` : 변환 PDF 및 결과 리포트 출력 폴더 (예시)
*   **[배치 이미지 안내]**: 윈도우 탐색기 내 `C:/IPLMS` 전체 디렉토리 트리 구조 캡처

#### [Slide 04] 설정 파일 구성 (1) - 폴더 및 LibreOffice 경로 설정
*   **슬라이드 제목**: 1.2. config.properties 설정 - 폴더 지정 및 엔진 경로
*   **필수 텍스트**:
    *   `properties/config.properties` 설정을 통한 기본 변환 경로 연동법
    *   **입력(input) 폴더 지정**: `converter.input.dir=C:\\IPLMS\\91_input` (역슬래시는 이중으로 지정 `\\`)
    *   **출력(output) 폴더 지정**: `converter.output.dir=C:\\IPLMS\\92_output`
    *   **LibreOffice 엔진 위치 지정**: `converter.libreoffice.path=C:\\IPLMS\\LibreOfficePortable\\LibreOfficePortable.exe`
*   **[배치 이미지 안내]**: 메모장 또는 VS Code로 편집 중인 `config.properties` 파일 내 폴더/경로 설정 부분 캡처

#### [Slide 05] 설정 파일 구성 (2) - REST API 및 시스템 주기 설정
*   **슬라이드 제목**: 1.3. config.properties 설정 - REST API 포트 및 타임아웃
*   **필수 텍스트**:
    *   **REST API 서비스 포트 정의**: `converter.server.port=8080` (사용 미선점 포트로 지정)
    *   **데몬 정기 변환 주기 설정 (분 단위)**: `daemon.interval.minutes=10`
    *   **단일 파일 변환 타임아웃 제한 (초 단위)**: `converter.timeout.seconds=90` (설정 시간 초과 시 강제 kill 후 에러 파일 생성)
*   **[배치 이미지 안내]**: `config.properties` 파일 하단부 포트 및 타이머 관련 주석과 코드 라인 캡처

### 📌 Chapter 2. 변환 엔진(Daemon) 실행 및 제어

#### [Slide 06] command 창에서 데몬 실행
*   **슬라이드 제목**: 2.1. CLI 환경에서의 변환 엔진 구동 (Daemon 실행)
*   **필수 텍스트**:
    *   콘솔창 및 백그라운드 환경 구동을 시작합니다.
    *   `C:/IPLMS/bin` 디렉토리로 이동 후 **`IPLMSDocsDaemon.cmd`** 배치 파일을 실행합니다.
    *   스크립트 실행 시 내부적으로 시스템 자바(Java JVM) 탐색, 환경 설정 무결성 검증, JAR 리소스 바인딩을 자동으로 수행합니다.
*   **[배치 이미지 안내]**: 명령 프롬프트(CMD) 창에서 `IPLMSDocsDaemon.cmd`를 기동하는 명령행 캡처

#### [Slide 07] GUI 실행화면 및 데몬 구동 제어
*   **슬라이드 제목**: 2.2. GUI 실행화면 제어 - 데몬 구동 및 중지
*   **필수 텍스트**:
    *   **데몬 구동 방법**:
        1.  프로그램 기동 시 기본 대기 상태로 활성화
        2.  상단 헤더의 **[실행]** 버튼 클릭
        3.  지정 주기에 따라 정기 폴더 스캐닝 가동 및 REST API 포트 바인딩 확인
    *   **데몬 종료 방법**:
        1.  상단 헤더의 **[종료]** 버튼 클릭
        2.  동작 중인 배치 스케줄러가 대기 상태로 전환되며 가동이 안전하게 중단됨
*   **[배치 이미지 안내]**: GUI 화면에서 [실행] 버튼이 비활성화되고 [종료] 버튼이 활성화된 상태의 가동 캡처

#### [Slide 08] 로그 지우기 및 안전 종료
*   **슬라이드 제목**: 2.3. GUI 실행화면 제어 - 로그 관리 및 시스템 완전 종료
*   **필수 텍스트**:
    *   **로그 지우기 방법**:
        *   콘솔 출력 버퍼가 누적되어 GUI 가독성이 저하될 경우 **[로그 지우기]** 버튼을 클릭하여 화면을 즉시 청소합니다. (실제 파일 로그나 API 동작에는 영향을 주지 않음)
    *   **프로그램 완전 종료**:
        *   우측 상단 닫기(`X`) 버튼 클릭 시, 구동 중인 내장 웹서버 프로세스 소켓을 안전하게 close하고 배치 스케줄러를 메모리에서 해제한 후 강제 리크 없이 프로세스를 다운시킵니다.
*   **[배치 이미지 안내]**: GUI 내 [로그 지우기] 버튼 영역 강조 캡처 및 창 닫기 경고 흐름도

### 📌 Chapter 3. 실시간 변환 서비스 (REST API) 연동

#### [Slide 09] REST API 구동 및 기본 가이드
*   **슬라이드 제목**: 3.1. REST API 실시간 변환 구동 사양
*   **필수 텍스트**:
    *   데몬 실행 시 내장 HttpServer가 함께 활성화되어, 파일 1개의 절대 경로를 넘겨받아 즉각적인 실시간 PDF 변환 처리를 지원합니다.
    *   **기본 엔드포인트**: `http://localhost:8080/api/convert`
    *   **지원 메소드**: `GET` (테스트용), `POST` (연계 시스템 인터페이스용)
    *   **CORS 기본 허용**: 타 도메인 웹브라우저 클라이언트 스크립트에서도 호출이 가능하도록 CORS 정책 탑재
*   **[배치 이미지 안내]**: GUI 콘솔 로그창 내 `>> [HttpServer] 내장 웹 서버가 포트 8080에서 실행 중입니다.` 메시지 표시 화면 캡처

#### [Slide 10] REST API 사용법 정의 (파라미터 규격)
*   **슬라이드 제목**: 3.2. REST API 파라미터 및 JSON 응답 규격
*   **필수 텍스트**:
    *   **요청 매개변수 (`filePath`)**: 변환하고자 하는 대상 파일의 로컬/서버 절대 경로 전달
    *   **GET 방식 (URL Query)**: `filePath=C:\temp\doc.docx`
    *   **POST 방식 (JSON Body)**: `{"filePath": "C:\\temp\\doc.docx"}`
    *   **반환 결과 (JSON)**:
        *   `status`: 성공 여부 (`success` / `error`)
        *   `pdfPath`: 변환된 PDF 결과 파일 절대 경로
        *   `txtPath`: 추출된 TXT 결과 파일 절대 경로
        *   `txtExtracted`: 텍스트 추출 완료 플래그 (`true` / `false`)
        *   `extractedTextContent`: 추출된 문서 텍스트 문자열
        *   `elapsedTime`: 변환 소요 시간 (초)
*   **[배치 이미지 안내]**: JSON 응답 구조 다이어그램 및 파라미터 설정표

#### [Slide 11] REST API 구동 및 결과 확인 예제
*   **슬라이드 제목**: 3.3. REST API 호출 테스트 및 결과 검증
*   **필수 텍스트**:
    *   `sample_request.cmd` 배치 파일을 이용한 CLI 환경 호출 실습
    *   **curl 명령을 이용한 실제 호출 커맨드**:
        *   `curl -i -X GET "http://localhost:8080/api/convert?filePath=C:\\IPLMS\\sample_doc.docx"`
    *   **결과 검증**:
        *   API 응답으로 HTTP `200 OK` 및 변환 정보 JSON 획득 확인
        *   원본 디렉토리 내에 물리적으로 `sample_doc.pdf`와 `sample_doc.txt`가 정상 생성되었는지 검증
*   **[배치 이미지 안내]**: CMD 콘솔창에서 curl을 통해 JSON 응답을 받아오는 전체 실행 결과 화면 캡처

*   **슬라이드 제목**: 4.1. GUI 실시간 렌더링 로그 항목 상세 설명

#### [Slide 13] 에러 상태 로그 및 트러블슈팅
*   **슬라이드 제목**: 4.2. 오류 리포트 및 트러블슈팅 가이드
*   **필수 텍스트**:
    *   **개별 파일 변환 실패시**:
        *   변환 불가 문서 발생 시 동일 폴더 내에 `{파일명}_ERR.txt` 에러 상세 기술 파일 생성
    *   **메모리 임계치(2GB) 초과 감지 시**:
        *   사용량이 2GB 도달 시 가비지 컬렉션(GC) 강제 작동 및 확보 실패 시 `SYSTEM_FATAL_ERROR.txt` 리포트 작성 후 엔진 자동 세이프 종료 조치
    *   **서버 포트 충돌**:
        *   내장 HTTP 포트 바인딩 에러 시 `ERROR: [HttpServer] 서버 시작 실패` 메시지가 검출되므로 포트 변경 필요
*   **[배치 이미지 안내]**: 92_output 폴더 내 생성된 `_ERR.txt` 내용 캡처 및 포트 바인딩 예외 콘솔 캡처
